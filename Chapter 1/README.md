# Workshop 1: การทำงานกับ Git และ GitHub

> เอกสารประกอบการสอนสำหรับฝึกใช้งาน **Git** เบื้องต้น ตั้งแต่การสร้างพื้นที่ทำงาน การบันทึกประวัติการทำงาน ไปจนถึงการอัปโหลดไฟล์ขึ้น Cloud/GitHub

---

## วัตถุประสงค์การเรียนรู้

หลังจากทำ Workshop นี้ ผู้เรียนจะสามารถ

- สร้างโฟลเดอร์สำหรับจัดเก็บงานได้อย่างเป็นระบบ
- เริ่มต้นใช้งาน Git ภายในโฟลเดอร์งานด้วยคำสั่ง `git init`
- เพิ่มไฟล์เข้าสู่ระบบติดตามของ Git ด้วยคำสั่ง `git add .`
- บันทึกประวัติการทำงานด้วยคำสั่ง `git commit`
- อัปโหลดงานขึ้น GitHub หรือ Cloud Repository ด้วยคำสั่ง `git push`

---

## เครื่องมือที่ต้องเตรียม

| เครื่องมือ | รายละเอียด |
|---|---|
| **Git** | โปรแกรมสำหรับจัดการเวอร์ชันของไฟล์งาน |
| **Terminal / Command Prompt** | ใช้พิมพ์คำสั่ง Git |
| **GitHub Account** | ใช้สำหรับอัปโหลดและจัดเก็บ Repository ออนไลน์ |
| **โฟลเดอร์งาน** | พื้นที่สำหรับเก็บไฟล์ของโปรเจกต์ |

---

## ขั้นตอนการทำงานกับ Git

### 1. สร้างโฟลเดอร์สำหรับเก็บงาน

สร้างโฟลเดอร์ใหม่เพื่อใช้เป็นพื้นที่ทำงานของโปรเจกต์ เช่น

```text
my-first-git-project
```

> แนะนำให้ตั้งชื่อโฟลเดอร์เป็นภาษาอังกฤษ และหลีกเลี่ยงการเว้นวรรค เพื่อให้ง่ายต่อการใช้งานผ่าน Terminal

---

### 2. เปิด Terminal หรือ Command Prompt

เปิด **Terminal** หรือ **Command Prompt (CMD)** แล้วไปยังตำแหน่งของโฟลเดอร์งานด้วยคำสั่ง

```bash
cd <ตำแหน่งโฟลเดอร์>
```

ตัวอย่าง

```bash
cd Desktop/my-first-git-project
```

---

### 3. เริ่มต้นใช้งาน Git ในโฟลเดอร์งาน

ใช้คำสั่งต่อไปนี้เพื่อสร้างพื้นที่ Git ภายในโฟลเดอร์งาน

```bash
git init
```

คำสั่งนี้จะทำให้ Git เริ่มติดตามการเปลี่ยนแปลงของไฟล์ในโฟลเดอร์นี้

---

### 4. เพิ่มไฟล์เข้าสู่ Git

เมื่อมีไฟล์งานอยู่ในโฟลเดอร์แล้ว ให้เพิ่มไฟล์ทั้งหมดเข้าสู่ Git ด้วยคำสั่ง

```bash
git add .
```

> เครื่องหมาย `.` หมายถึง เพิ่มไฟล์ทั้งหมดที่มีการเปลี่ยนแปลงในโฟลเดอร์ปัจจุบัน

---

### 5. บันทึกประวัติการทำงาน

เมื่อทำงานเรียบร้อยแล้ว ให้บันทึกประวัติการทำงานด้วยคำสั่ง

```bash
git commit -m "ข้อความอธิบายการเปลี่ยนแปลง"
```

ตัวอย่าง

```bash
git commit -m "Initial commit"
```

ข้อความ commit ควรเขียนให้สั้น กระชับ และสื่อความหมาย เช่น

```bash
git commit -m "Add workshop instruction"
```

---

### 6. อัปโหลดงานขึ้น Cloud หรือ GitHub

หลังจากเชื่อมต่อ Repository บน GitHub แล้ว สามารถอัปโหลดงานขึ้น Cloud ด้วยคำสั่ง

```bash
git push
```

หากเป็นการ push ครั้งแรก อาจต้องใช้คำสั่งลักษณะนี้

```bash
git push -u origin main
```

## สรุปคำสั่ง Git ที่ใช้ใน Workshop

| ลำดับ | คำสั่ง | ความหมาย |
|---:|---|---|
| 1 | `cd <ตำแหน่งโฟลเดอร์>` | ไปยังตำแหน่งโฟลเดอร์งาน |
| 2 | `git init` | เริ่มต้นใช้งาน Git ในโฟลเดอร์ |
| 3 | `git add .` | เพิ่มไฟล์ทั้งหมดเข้าสู่ Git |
| 4 | `git commit -m "ข้อความ"` | บันทึกประวัติการทำงาน |
| 5 | `git push` | อัปโหลดงานขึ้น Cloud/GitHub |

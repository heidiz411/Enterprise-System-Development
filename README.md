## 🎯 จุดมุ่งหมายของรายวิชา

รายวิชานี้ออกแบบเพื่อให้นักศึกษาสามารถเรียนรู้และฝึกปฏิบัติการพัฒนาระบบซอฟต์แวร์ในบริบทองค์กร โดยเน้นทั้งความรู้ทางเทคนิค กระบวนการทำงานเป็นทีม และจรรยาบรรณวิชาชีพ

1. เพื่อให้นักศึกษาสามารถเลือกใช้เครื่องมือและวิธีการที่สำคัญในการพัฒนาซอฟต์แวร์สำหรับองค์กรได้อย่างเหมาะสม เช่น **Git**, **Docker** และ **Cloud Platform**
2. เพื่อให้นักศึกษาสามารถทำงานร่วมกันเป็นทีมตามกระบวนการพัฒนาซอฟต์แวร์ขององค์กร ตั้งแต่การจัดการซอร์สโค้ด การทบทวนโค้ด การทดสอบ ไปจนถึงการนำระบบขึ้นใช้งาน
3. เพื่อให้นักศึกษาสามารถวิเคราะห์ปัญหา ออกแบบแนวทางแก้ไข และพัฒนาระบบต้นแบบที่สอดคล้องกับบริบทการทำงานจริงของสถานประกอบการ
4. เพื่อส่งเสริมความรับผิดชอบ คุณธรรม จริยธรรม จรรยาบรรณวิชาชีพ ความใฝ่รู้ และความอดทนในการแก้ไขปัญหาทางเทคนิค

## 🧭 คำอธิบายรายวิชา

ศึกษาและฝึกปฏิบัติเกี่ยวกับเครื่องมือและวิธีการที่สำคัญสำหรับการพัฒนาระบบในองค์กร เช่น **Version Control System** โดยใช้ Git, **Container Technology** โดยใช้ Docker และการใช้งานระบบ **Cloud** รวมถึงการฝึกพัฒนาและบริหารจัดการระบบด้วยเครื่องมือที่ใช้จริงในองค์กร

**English Description**  
Tools and techniques for enterprise software development including version control systems such as Git, containers like Docker, and cloud systems; hands-on experience in software development using commonly used tools and techniques in enterprise environments.

## 🧩 แนวคิดการจัดการเรียนรู้แบบ CWIE

รายวิชานี้เน้นการเรียนรู้จากสถานการณ์จำลองการทำงานจริงในองค์กร โดยให้นักศึกษาเรียนรู้ผ่านการปฏิบัติจริงในลักษณะทีมพัฒนาซอฟต์แวร์

### กิจกรรม CWIE สำคัญ

- ใช้ **Git / Git Workflow** เพื่อจัดการซอร์สโค้ดร่วมกัน
- ใช้ **Remote Repository** และกระบวนการ Pull Request / Merge Request
- ฝึก **Code Review** และการให้ข้อเสนอแนะเชิงสร้างสรรค์
- ใช้ **Issue Tracking / Milestone / Kanban Board** เพื่อบริหารงานทีม
- ใช้ **Docker / Docker Compose** เพื่อจำลองสภาพแวดล้อมการทำงานของระบบ
- ทดลองนำระบบขึ้นใช้งานบน **Cloud Platform** หรือแพลตฟอร์มจำลอง
- ออกแบบกระบวนการ **CI/CD Pipeline** สำหรับ build, test และ deploy
- ทำโครงงานจำลองการพัฒนาระบบในองค์กร ตั้งแต่ requirement, architecture, development, deployment และ presentation

## 🚀 โครงงานปลายภาคแบบ CWIE

นักศึกษาจะทำงานเป็นทีมเพื่อพัฒนาระบบต้นแบบในบริบทองค์กร โดยต้องแสดงหลักฐานการทำงานผ่าน repository และกระบวนการที่เป็นระบบ

### ผลงานที่คาดหวัง

- Repository ที่มีประวัติการพัฒนาอย่างต่อเนื่อง
- README ของโครงงานที่อธิบายวิธีติดตั้งและใช้งานระบบ
- Issue Board หรือหลักฐานการแบ่งงานภายในทีม
- Dockerfile และ/หรือ docker-compose.yml สำหรับรันระบบ
- Deployment Report หรือหลักฐานการนำระบบขึ้นใช้งาน
- สไลด์หรือรายงานสรุปการพัฒนา ปัญหา วิธีแก้ และบทเรียนที่ได้รับ
- การนำเสนอและสาธิตระบบต้นแบบ

### ตัวอย่างโครงสร้าง repository สำหรับโครงงาน

```text
project-name/
├── README.md
├── docs/
│   ├── requirements.md
│   ├── architecture.md
│   └── deployment-report.md
├── src/
│   └── ...
├── tests/
│   └── ...
├── Dockerfile
├── docker-compose.yml
├── .env.example
└── .github/
    └── workflows/
        └── ci.yml
```

## ✅ แนวปฏิบัติสำหรับนักศึกษา

- เข้าเรียนและปฏิบัติตามข้อตกลงของรายวิชาอย่างสม่ำเสมอ
- ส่งงานตามเวลาที่กำหนด และแสดงหลักฐานการทำงานผ่าน repository
- ไม่คัดลอกผลงานผู้อื่น และต้องอ้างอิงแหล่งที่มาของโค้ดหรือเอกสารที่นำมาใช้
- ไม่เปิดเผย credential, token, password หรือข้อมูลสำคัญใน repository
- ทำงานร่วมกับทีมอย่างรับผิดชอบ เคารพความคิดเห็นของผู้อื่น และให้ข้อเสนอแนะอย่างสร้างสรรค์
- ฝึกอ่าน log, วิเคราะห์ปัญหา และแก้ไขปัญหาอย่างเป็นระบบ
